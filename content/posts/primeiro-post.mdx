# Meu primeiro post em MDX

Este é o **primeiro post funcional** da nova estrutura do site, agora com suporte a MDX.

Agora a base já está preparada para você:

- versionar tudo no GitHub;
- publicar automaticamente no Cloudflare Pages;
- escrever conteúdo em arquivos `.mdx`;
- embutir componentes React diretamente nos posts.

<MdxCallout title="Exemplo de componente MDX">
  Você pode destacar blocos importantes com componentes React simples, sem sair do fluxo de escrita do post.
</MdxCallout>

## O que passou a ser possível com MDX

Antes, com Markdown puro, o post ficava limitado a texto, imagens e blocos estáticos. Agora eu posso misturar conteúdo com componentes e JSX no mesmo arquivo.

### 1. Callouts reutilizáveis

Este bloco abaixo é um componente React renderizado dentro do post:

<MdxCallout title="Feature real de MDX">
  Este destaque não é um blockquote estilizado. Ele vem de um componente registrado em
  {' '}<code>components/mdx-components.tsx</code>{' '}
  e pode evoluir com variações de layout, ícones ou props.
</MdxCallout>

### 2. Renderização dinâmica com JSX

Também dá para gerar trechos da interface com JavaScript direto no conteúdo:

<div className="tag-list">
  {['Componente customizado', 'JSX no conteúdo', 'Links internos inteligentes'].map((feature) => (
    <span key={feature} className="tag">
      {feature}
    </span>
  ))}
</div>

O trecho acima seria inviável em Markdown puro sem transformar tudo em HTML manual ou mover a lógica para fora do post.

### 3. Navegação interna mais rica

Como o renderer do MDX intercepta links, eu também posso apontar para páginas do próprio site como
{' '}[a listagem completa do blog](/blog){' '}
sem perder o comportamento de navegação do app.

## Exemplo de bloco de código

```python
import pandas as pd

vendas = pd.DataFrame(
    {
        "mes": ["jan", "fev", "mar"],
        "valor": [120, 155, 141],
    }
)

print(vendas)
```

## Como criar o próximo post

Crie um novo arquivo em `content/posts`, por exemplo:

```bash
content/posts/segundo-post.mdx
```

Use este modelo:

```mdx
---
title: "Título do post"
description: "Descrição curta para cards e SEO"
date: "2026-04-10"
tags:
  - python
  - analytics
---

# Título do post

Texto normal em MDX.

<MdxCallout title="Dica">
  Você pode reutilizar componentes registrados em `components/mdx-components.tsx`.
</MdxCallout>

<div className="tag-list">
  {['MDX', 'React', 'Conteúdo'].map((item) => (
    <span key={item} className="tag">
      {item}
    </span>
  ))}
</div>
```
