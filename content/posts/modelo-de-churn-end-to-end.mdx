# Modelo de churn end-to-end para assinatura digital

Este é um **projeto fictício de portfólio** criado para demonstrar como um case completo pode aparecer na nova seção de projetos.

## Contexto

Uma empresa de assinatura digital percebeu aumento da evasão entre clientes recorrentes no plano mensal. O objetivo era antecipar o risco de cancelamento e identificar alavancas de retenção.

## Objetivo

- prever a probabilidade de churn no horizonte de 30 dias;
- entender os principais fatores associados à evasão;
- propor uma priorização operacional para campanhas de retenção.

## Base de dados

O projeto considera uma base consolidada com:

- histórico de assinaturas;
- eventos de uso do produto;
- chamados de suporte;
- dados transacionais;
- atributos cadastrais e segmentação do cliente.

## Pipeline analítico

### 1. Preparação e exploração

Foram criadas variáveis de comportamento para capturar intensidade e qualidade de uso:

- frequência semanal de acesso;
- tempo desde o último uso;
- número de tickets no suporte;
- queda recente de engajamento;
- atraso ou falha em pagamentos.

```python
features = [
    "acessos_7d",
    "dias_sem_login",
    "tickets_30d",
    "queda_engajamento",
    "falha_pagamento",
]
```

### 2. Modelagem

Foi usada uma abordagem supervisionada com regressão logística como baseline e gradient boosting como modelo principal.

```python
from sklearn.metrics import roc_auc_score
from lightgbm import LGBMClassifier

model = LGBMClassifier(
    n_estimators=300,
    learning_rate=0.05,
    num_leaves=31,
    random_state=42,
)

model.fit(X_train, y_train)
score = roc_auc_score(y_valid, model.predict_proba(X_valid)[:, 1])
print(round(score, 4))
```

### 3. Resultados simulados

- AUC em validação: **0.84**
- top decile lift: **3.1x**
- recall no grupo priorizado: **58%**

## Principais achados

- queda abrupta de uso nas últimas duas semanas foi o sinal mais forte de churn;
- clientes com falhas recentes de pagamento apresentaram risco muito maior;
- tickets de suporte sem resolução rápida aumentaram a propensão à evasão.

## Proposta de uso

O modelo seria operacionalizado em lotes diários com segmentação em três faixas:

- **alto risco**: ação humana com oferta e contato direcionado;
- **médio risco**: automação com conteúdo educativo e incentivo de uso;
- **baixo risco**: monitoramento sem intervenção imediata.

## Próximos passos

- teste controlado da régua de retenção;
- calibragem por segmento de cliente;
- acompanhamento de impacto financeiro e redução real de churn.
